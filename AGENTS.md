# Projet MMO Fantastique

## Stack Technique
- **Client**: Godot Engine 4.x (GDScript/C#)
- **Serveur**: Go 1.21+
- **Base de données**: À définir
- **Réseau**: WebSocket/TCP

## Équipes

### Leadership
- `@ceo` - Vision, roadmap, décisions stratégiques
- `@cto` - Architecture technique, choix technologiques

### Développement
- `@dev-team` - Implémentation client/serveur
- `@qa-team` - Tests et assurance qualité
- `@secu-team` - Sécurité et anti-cheat
- `@design-team` - Game design et UX

### Coordination
- `@meetings` - Organisation des réunions

## Utilisation

### Démarrer un projet
```
/init
```

### Exemple de workflow
1. **Planification**: `@ceo` définit la roadmap
2. **Design technique**: `@cto` + `@design-team` spécifient les features
3. **Développement**: `@dev-team` implémente
4. **Tests**: `@qa-team` valide
5. **Sécurité**: `@secu-team` audite
6. **Réunion**: `@meetings` organise les syncs

### Réunions typiques
- **Sprint Planning**: CEO + toutes les équipes
- **Design Review**: Design + Dev + CTO
- **Security Audit**: Secu + CTO + Dev
- **Demo**: QA + CEO + toutes les équipes

## Règles
- Tous les agents utilisent le modèle Qwen
- Travail parallèle autorisé entre équipes
- Validation CEO pour les décisions de scope
- Validation CTO pour les décisions techniques
- Approval manuelle requise pour les edits (sauf QA/Dev en bash)
